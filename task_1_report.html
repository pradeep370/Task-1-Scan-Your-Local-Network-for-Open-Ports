SCAN YOUR LOCAL NETWORK FOR OPEN PORTS

OBJECTIVE
To scan my local network for open ports using Nmap and understand which services are exposed, practicing basic network reconnaissance techniques.

Date - 04 august 2025
Submitted by - Pradeep s

TOOLS ARE USED BY ME
* Nmap - for port scanning
* Command Prompt/terminal - to find Local IP address
* Wireshark - to capture network traffic during scan

STEPS WHICH I FOLLOWED FOR ENVIRONMENT SETUP
* I installed a vulnerable Windows machine as the target system inside Oracle VirtualBox.
* Used Kali Linux as the attacker machine, also running inside the VirtualBox.
* I Configured both machines to use a Bridged Adapter network.

FIND LOCAL IP ADDRESSES
* I found local IP addresses using:
ipconfig (on Windows) and ifconfig (on Kali Linux).

* The Identified Ip addresses of Attacker and Target machine is,
Kali Linux (Attacker machine):192.168.1.9
Windows (target Machine):192.168.1.8

(Router/Modem): 192.168.1.1
(Windows/Main os): 192.168.1.7

* Subnet Mask (Both Systems):255.255.255.0

* Before starting the scan, I tested the interconnectivity between Kali Linux and the Windows VM using the ping command.

* After I Run TCP SYN scan using this Command:

nmap -sS 192.168.1.8/24
